## Overview

Caronte V1 is the flight computer firmware for a two-stage experimental sounding rocket targeting 50–100 km apogee. It runs on a **WeAct STM32F405RGT6** and implements a full avionics stack including sensor fusion, active aerodynamic control, telemetry, and flight state management.

## Hardware

| Component | Part | Interface |
|-----------|------|-----------|
| Microcontroller | WeAct STM32F405RGT6 | — |
| Primary IMU | GY-601N1 ICM-45686 | I2C @ 0x68 |
| High-G IMU | CJMCU ADXL375 | I2C @ 0x53 |
| Barometer | MS5611-01BA03 | I2C @ 0x77 |
| GPS | NEO-6M | UART1 |
| Radio | Ebyte E32-900T20D (LoRa) | UART6 |
| Servos | 4× fin control servos | PWM |
| Pyros | 4× channels (TIP41C) | GPIO |
| Camera | RunCam Split 4 | UART |
| Storage | MicroSD (SDIO) | SDIO |

## Firmware Architecture

```
Caronte_V1.ino          Main loop — 100 Hz control cycle
├── config.h            Pin mapping, addresses, tuning constants
│
├── Estimator           Full INS stack
│   ├── IMU_Calibration 6-position gyro/accel calibration (Flash)
│   ├── EKF9            9-state Extended Kalman Filter
│   │   └── States: alt, vz, roll, pitch, yaw, gyro_bias(3), accel_bias
│   └── GPS_INS         2-state horizontal Kalman (vN, vE)
│
├── FlightFSM           Flight state machine
│   └── PAD → POWERED_1 → STAGING → POWERED_2 →
│       COASTING → APOGEE → DESCENT → LANDED
│
├── PIDController       3-axis PID (Roll / Pitch / Yaw)
├── ServoController     Fin deflection with aero LUT inversion
├── aero_table.h        CL(α) lookup table — generated by AeroSim Analyzer
│
├── Sensors
│   ├── ICM45686        Primary IMU driver (I2C)
│   ├── ADXL375         High-G IMU driver (I2C) — handoff at 18g
│   ├── MS5611          Barometric altimeter driver
│   └── GPS_NEO6M       NMEA parser + INS feed
│
├── LoRaTelemetry       Binary telemetry protocol (LoRa E32)
├── DataLogger          SD card logging (SDIO)
├── TestMode            Pre-flight hardware verification
├── Buzzer / RunCam / LED_Utils   Peripheral drivers
└── IWatchdog           Hardware watchdog (500 ms timeout)
```

## Key Features

- **Dual-IMU architecture** — ICM-45686 for nominal flight, automatic handoff to ADXL375 (±200g) when acceleration exceeds 18g (hysteresis back at 15g)
- **EKF9** — 9-state EKF with adaptive measurement noise, GPS altitude fusion, and barometric update at 10 Hz
- **Active fin control** — PID + aerodynamic inversion via CFD-generated LUT (AeroSim Analyzer)
- **ZUPT** — Zero-velocity update on the pad for gyro bias refinement
- **LoRa telemetry** — 5 packet types + 6 command types including in-flight PID gain update
- **Watchdog** — IWDG hardware watchdog, 500 ms timeout

## Build & Flash

### Requirements
- Arduino IDE 2.x
- [STM32duino](https://github.com/stm32duino/Arduino_Core_STM32) board package (2.x)
- STM32CubeProgrammer (for DFU upload)

### Board Settings (Arduino IDE)

| Setting | Value |
|---------|-------|
| Board | Generic STM32F4 series |
| Board part number | Generic F405RGTx |
| Upload method | STM32CubeProgrammer (DFU) |
| USB support | CDC (generic Serial) |
| Optimize | Smallest (-Os default) |

### Flash via DFU
1. Hold **BOOT0**, press and release **RESET**, release **BOOT0**
2. Click **Upload** in Arduino IDE

## Configuration

All tunable parameters are in `config.h`:

```cpp
// Saturation thresholds
#define ICM_SATURATION_MS2  176.52f   // 18g — switch to ADXL375
#define ICM_RESUME_MS2      147.10f   // 15g — return to ICM

// Control limits
#define DELTA_MAX_DEG       15.0f     // Maximum fin deflection

// Watchdog
#define WATCHDOG_TIMEOUT_US 500000UL  // 500 ms
```

## Telemetry Protocol

Binary protocol over LoRa E32. Frame: `[0xCA][TYPE][SEQ][PAYLOAD]`

| Packet | Type | Description |
|--------|------|-------------|
| STATUS | 0x01 | Phase, armed, sensors, EKF health |
| ATTITUDE | 0x02 | Roll, pitch, yaw, angular rates |
| POSITION | 0x03 | Altitude, velocity, GPS, air density |
| PYRO_EVENT | 0x04 | Channel fired, flight time |
| HEARTBEAT | 0x05 | RSSI, uptime |

| Command | Code | Description |
|---------|------|-------------|
| ARM | 0xA1 | Arm pyros |
| DISARM | 0xA2 | Disarm |
| FORCE_PHASE | 0xA3 | Debug phase override |
| SET_GAINS | 0xA4 | In-flight PID update |
| ZUPT_COMMIT | 0xD3 | Commit gyro bias (RAM or Flash) |

## Aerodynamic Table

`aero_table.h` is generated by **AeroSim Analyzer** after CFD analysis in SimScale:

1. Run `simscale_sweep_gui.py` — parametric CFD sweep (α, V, ρ)
2. Load CSV in `aero_analysis_gui.py` — fit CL(α) surface
3. Export `aero_table.h` — CL→AOA lookup table (N points, linear interpolation)

The placeholder table uses a linear approximation `CL = 0.055 × α`. **Replace before flight.**
